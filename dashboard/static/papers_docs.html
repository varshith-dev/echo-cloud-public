OQENS Papers Documentation

Welcome to the official documentation for OQENS Papers. Papers is a lightning-fast, distraction-free Markdown workspace designed for creators, developers, and writers who want to craft beautiful documents and publish them to the web instantly.

Note: OQENS Papers uses GitHub Flavored Markdown (GFM) out of the box, meaning almost all standard formatting from GitHub and other tools will work perfectly here.

Markdown Basics

If you've never used Markdown before, it's a lightweight markup language that you can use to add formatting elements to plaintext text documents.

Headings & Text

# Heading 1
## Heading 2
### Heading 3

**Bold text**
*Italic text*
~~Strikethrough~~

Lists and Links

- Bullet point 1
- Bullet point 2
  - Nested point

1. Numbered item 1
2. Numbered item 2

[Click here for OQENS](https://oqens.me)

Math & Equations

OQENS Papers natively supports rendering LaTeX mathematical expressions using KaTeX. This is perfect for technical writers and academics.

Use single dollar signs $...$ for inline equations, and double dollar signs $$...$$ for block equations.

Here is an inline equation: $E=mc^2$

Here is a block equation:
$$
\frac{n!}{k!(n-k)!} = \binom{n}{k}
$$

Interactive Charts (Chart.js)

You can embed fully interactive, responsive charts directly into your documents without writing any JavaScript logic. We support this by parsing special JSON blocks within your markdown.

To create a chart, create a code block with the language set to chartjs and paste standard Chart.js JSON configuration inside.

```chartjs
{
  "type": "bar",
  "data": {
    "labels": ["Jan", "Feb", "Mar", "Apr"],
    "datasets": [{
      "label": "Revenue",
      "data": [12000, 19000, 3000, 5000],
      "backgroundColor": "#3b82f6"
    }]
  }
}
```

Mermaid Diagrams

Mermaid is a JavaScript-based diagramming and charting tool that renders Markdown-inspired text definitions dynamically to create and modify diagrams.

Just use the mermaid code block tag.

```mermaid
graph TD;
    A[Client]-->B[OQENS Edge CDN];
    B-->C[Application Server];
    C-->D[(PostgreSQL)];
```

Advanced Data Tables (Grid.js)

Standard Markdown tables are supported, but if you need pagination, sorting, or searching for large datasets, you can use our native Grid.js integration.

```gridjs
{
  "search": true,
  "sort": true,
  "pagination": { "limit": 3 },
  "columns": ["Name", "Email", "Status"],
  "data": [
    ["John Doe", "john@example.com", "Active"],
    ["Jane Smith", "jane@example.com", "Pending"],
    ["Bob Wilson", "bob@example.com", "Active"]
  ]
}
```

Publishing to the Web

Every document in Papers is private to your workspace by default. To publish a document to the world:

  • Click the Publish button in the top right corner of the editor.
  • Set a custom slug (e.g., /my-awesome-post).
  • Toggle visibility to "Public".

Your document will be instantly built, cached globally on our CDN, and accessible at papers.oqens.me/p/{username}/{slug} (or on your custom domain if configured).

SEO & Meta Information

When you publish a page, Papers automatically generates meta tags for search engines and social media previews. However, you can explicitly define these at the very top of your document using YAML frontmatter:

---
title: "The Ultimate Guide to Markdown"
description: "Learn how to use OQENS Papers to write faster."
image: "https://example.com/banner.jpg"
---
